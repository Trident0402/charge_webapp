# 離線資產管理 Web App Implementation Plan

## 1. 目標

建立一個 iPhone 和 Android 都可以使用的離線 Web App，用來管理：

- 各銀行帳戶
- Line Pay Money
- 錢包
- 股票帳戶
- 預期入帳金額

第一版只使用：

- HTML
- CSS
- JavaScript
- localStorage
- PWA manifest
- Service Worker

不使用：

- 後端
- React / Vue / Angular
- Node 打包工具
- 雲端同步
- 登入系統

## 2. 簡化架構原則

檔案不要拆太細。同一個主題放在同一個檔案裡，資料夾最多只保留一層。

例如：

- 帳戶相關：放 `js/accounts.js`
- 收入支出相關：放 `js/transactions.js`
- 股票相關：放 `js/stocks.js`
- 預期入帳相關：放 `js/expected-income.js`
- 設定與備份：放 `js/settings.js`

不要再拆成：

```text
features/accounts/accounts.logic.js
features/accounts/accounts.render.js
features/accounts/accounts.form.js
```

這樣對目前的小型 App 太複雜。

負責一般帳戶收入與支出：

- 新增收入
- 新增支出

- 刪除收入 / 支出
- 帳戶間轉帳
- 修改帳戶間轉帳
- 刪除帳戶間轉帳
- 顯示交易紀錄
- 依年月篩選交易紀錄
- 處理收支表單

轉帳設計：

- 在一般帳戶詳情頁右下角 `+` 選單中提供「轉帳」
- 使用者選擇日期、原因、金額、轉入帳戶
- 只在一般資產帳戶間轉帳，股票帳戶不納入轉帳目標
- 儲存時建立兩筆交易：
  - 目前帳戶：`transfer-out`
  - 轉入帳戶：`transfer-in`
- 兩筆交易使用同一個 `transferId` 方便未來追蹤
- 修改轉帳時，要同步更新同一個 `transferId` 的兩筆紀錄
- 刪除轉帳時，要同步刪除同一個 `transferId` 的兩筆紀錄

### js/stocks.js

- 修改買入 / 賣出紀錄
- 刪除買入 / 賣出紀錄
- 修改每日市價
- 刪除每日市價
- 記住已輸入過的股票代號與股票名稱
- 計算持股數
- 計算平均成本
- 計算總成本
- 取得最新市價
- 計算目前市值
- 計算未實現損益
- 計算報酬率
- 顯示股票帳戶詳情

股票帳戶操作設計：

- 股票帳戶詳情頁右下角顯示 `+`
- 點擊後打開簡單動作選單
- 選單包含：
  - 買入
  - 賣出
  - 新增市價

股票代號與名稱記憶：

- 從 `stockTrades` 中整理已輸入過的股票代號與股票名稱
- 下一次新增股票交易時，股票代號與股票名稱欄位可以從歷史清單選擇
- 選到股票代號時，自動帶入對應名稱
- 選到股票名稱時，自動帶入對應代號

股票持股統合：

- 同一股票代號相同，視為同一持股
- 若股票代號不同但股票名稱相同，也視為同一持股
- 統合後加總股數、總成本、市值
- 平均成本 = 統合後總成本 / 統合後持股數
- 最新市價優先使用最新日期的市價紀錄
- 若沒有市價紀錄，使用最新一筆買入價格作為市價輔助

## 7. 計算邏輯

### 一般帳戶餘額

```text
餘額 = 初始金額 + 收入總額 - 支出總額 + 轉入總額 - 轉出總額
```

### 股票持股數

```text
持股數 = 買入股數總和 - 賣出股數總和
```

### 股票平均成本

```text
平均成本 = 目前總成本 / 目前持股數
```

### 股票市值

```text
股票市值 = 持股數 * 最新手動市價
```

如果沒有市價紀錄：

```text
最新市價 = 平均成本
```

### 股票未實現損益

```text
未實現損益 = 股票市值 - 總成本
```

### 股票報酬率

```text
報酬率 = 未實現損益 / 總成本 * 100
```

### 總資產

```text
總資產 = 一般帳戶餘額總和 + 股票帳戶市值總和
```

- 股票帳戶不使用這個收支月篩選

## 12. 近期修正計畫

### 12.1 紀錄修改與刪除

- 一般收支、轉帳、股票買賣、股票市價、預期入帳的列表紀錄，改成點擊整個紀錄方框進入修改頁。
- 列表上不另外放「修改 / 刪除」小按鈕，避免手機畫面太擠。
- 刪除動作放在各自的修改頁面底部，使用者進入該筆資料後再刪除。

### 12.2 股票持股收合顯示

- 持股區塊使用可展開 / 收合設計。
- 未展開時盡量壓縮成兩行，主要顯示股票識別、股數、未實現損益、損益率。
- 展開後顯示最新市價、均價、總市值、總成本。
- 展開內容不顯示估計稅費，避免畫面資訊過多。

### 12.3 未實現損益計算

- 每一檔持股的未實現損益需扣除預估賣出相關成本。
- 預估成本公式：

```text
預估扣除成本 = 目前市值 * (交易稅 0.3% + 券商手續費 0.1425% * 2)
```

- 持股未實現損益公式：

```text
未實現損益 = 總市值 - 總成本 - 預估扣除成本
```

- 股票帳戶上方的未實現損益，直接加總所有持股的未實現損益，不再用總市值減總成本重新計算。

### 12.4 餘額調整

- 一般帳戶右下角 `+` 選單移除「餘額調整」。
- 新增 / 修改收支表單不再提供「調整」作為可選類型。
- 若舊資料裡已存在調整紀錄，仍可透過點擊紀錄進入修改頁並刪除。

### 12.5 初始預設帳戶

- 第一次開啟 App、或清除資料後重新建立初始資料時，先建立常用資產帳戶。
- 預設帳戶如下：
  - 玉山銀行：銀行帳戶
  - 富邦銀行：銀行帳戶
  - Line pay money：Line Pay Money
  - 現金：錢包
  - 富邦證卷：證卷帳戶
- 預設初始金額皆為 `0`。
- 已經有 localStorage 舊資料的使用者，不自動覆蓋或新增這些帳戶，避免產生重複資料。

### 12.6 設定頁補齊基礎帳戶

- 設定頁新增「新增基礎帳戶」按鈕。
- 按下後檢查目前帳戶清單，補齊以下五個基礎帳戶：
  - 玉山銀行：銀行帳戶
  - 富邦銀行：銀行帳戶
  - Line pay money：Line Pay Money
  - 現金：錢包
  - 富邦證卷：證卷帳戶
- 若已存在相同名稱或相同預設 ID 的帳戶，就視為已存在，不再新增。
- 如果使用者已經有重複帳戶，系統不自動合併、不刪除、不更動既有資料。
- 執行後顯示本次新增幾個帳戶；若五個都已存在，提示已不需新增。

### 12.8 預期入帳頁手機化

- 預期入帳列表頁只保留：
  - 預期入帳摘要
  - 預期入帳清單
- 新增表單不直接顯示在列表頁，改由右下角浮動 `+` 按鈕開啟，與其他頁面操作位置一致。
- 點擊既有預期入帳紀錄方框時，進入修改頁。
- 修改頁保留刪除按鈕，符合其他紀錄「點卡片修改、刪除在修改頁」的模式。
- 首頁摘要中的預期入帳欄位文字統一為「預期入帳總額」。

### 12.9 預期入帳全資料統計

- App 內預期入帳相關標題不再加上月份限定文字，統一改成「預期入帳」。
- 預期入帳總額不再只計算特定月份，改為統計所有預期入帳資料。
- 首頁「預期入帳總額」要顯示所有預期入帳的總額。
- 預期入帳頁摘要也要顯示所有預期入帳的總額、已入帳、尚未入帳與筆數。
- 新增、修改、刪除、標記已入帳後，首頁與預期入帳頁都要能反映最新統計。

### 12.10 預期入帳清單範圍與卡片密度

- 預期入帳頁的摘要仍統計所有預期入帳資料。
- 預期入帳清單只顯示連續兩個月資料：
  - 使用者選定月份
  - 選定月份的下一個月
- 月份選項放在清單區塊左上角，使用 `<input type="month">`。
- 預設月份為目前年月。
- 新增或修改預期入帳後，若該筆日期不在目前清單範圍，清單月份切換到該筆資料月份。
- 預期入帳清單卡片中直接顯示備註文字。
- 每張卡片的「標記已入帳 / 改為未入帳」按鈕縮小並放在方框右下角，避免佔用整行。

### 12.11 收支分類 Tag

- 收入 / 支出表單的「分類」欄位改成 tag 輸入。
- 基礎分類 tag 包含：
  - 薪資

### 12.15 虛擬貨幣持倉帳戶

- 此規劃取代 `12.14` 的一般資產帳戶簡化版本。
- 將虛擬貨幣帳戶從一般資產帳戶升級成類似股票帳戶的獨立帳戶頁。
- 新增 `js/crypto.js`，集中處理虛擬貨幣帳戶的交易、市價、匯率、持倉與表單事件。
- 新增資料欄位：
  - `cryptoTrades`：買入 / 賣出紀錄。
  - `cryptoPrices`：每日市價紀錄。
- 虛擬貨幣交易欄位：
  - 買入 / 賣出
  - 幣種代號
  - 幣種名稱
  - 數量
  - 美元單價
  - 台幣美金匯率
  - 手續費（美元）
  - 日期
- 虛擬貨幣市價欄位：
  - 幣種代號
  - 美元市價
  - 台幣美金匯率
  - 日期
  - 備註
- 持倉計算：
  - 同樣的幣種代號或名稱統合成一個持倉。
  - 買入用「美元單價 × 匯率」換算成台幣成本。
  - 手續費用「美元手續費 × 匯率」加入台幣成本。
  - 賣出依照目前平均成本扣減剩餘成本。
  - 最新市價優先使用市價紀錄的最新美元價與匯率；沒有市價紀錄時使用最新買入價格與匯率；再沒有則使用平均成本反推。
- 畫面呈現：
  - 虛擬貨幣帳戶頁使用類似股票帳戶的 summary、持倉、買賣紀錄、市價紀錄。
  - 持倉未展開時顯示數量、未實現損益、報酬率；展開後顯示美元市價、匯率、台幣市值、台幣成本與平均成本。
  - 新增買入、賣出、新增市價都放在右下角 `+` 操作選單。
- 首頁統計：
  - 總資產加入虛擬貨幣台幣市值。
  - 首頁小統計新增虛擬貨幣市值。
- 帳戶刪除時同步刪除該帳戶的虛擬貨幣交易與市價紀錄。
- PWA 離線快取加入 `js/crypto.js` 並提升快取版本。

### 12.17 月報表子頁選單與實現損益月曆

- 將月報表頂部「圓餅圖 / 月曆」切換改成與頁面內容連在一起的 tab 樣式。
- 圓餅圖子頁不再使用大型「支出 / 收入」segmented control，改成子頁內的簡約三條線下拉式選單。
- 月曆子頁也加入同樣的三條線下拉式選單。
- 圓餅圖子頁選單項目：
  - 支出
  - 收入
- 月曆子頁選單項目：
  - 收入支出
  - 收入
  - 支出
  - 股票實現損益
  - 虛擬貨幣實現損益
- 股票實現損益計算：
  - 依股票帳戶買賣紀錄排序。
  - 買入增加股數與台幣成本。
  - 賣出依目前平均成本計算成本基礎。
  - 實現損益 = 賣出金額 - 成本基礎 - 賣出手續費 - 賣出稅費。
  - 結果依賣出日期放入月曆。
- 虛擬貨幣實現損益計算：
  - 依虛擬貨幣帳戶買賣紀錄排序。
  - 買入用美元價格與匯率換算台幣成本。
  - 賣出用美元價格與匯率換算台幣收入。
  - 實現損益 = 賣出台幣收入 - 成本基礎 - 手續費美元換算台幣。
  - 結果依賣出日期放入月曆。
- 月曆格子依選單模式顯示對應數值，損益可顯示正負值。
