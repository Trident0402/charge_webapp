# 月帳務報表 Implementation Plan

## 1. 目標

在底部導覽列的「預期入帳」與「設定」之間新增一個「月帳務報表」頁面，用手機 App 的方式呈現收入 / 支出的分類統計。

第一版已依此計畫開始實作，實作範圍以純前端 HTML / CSS / JS 為主。

## 2. 導覽位置

底部導覽列調整為：

```text
首頁 / 預期入帳 / 月報表 / 設定
```

新增頁面名稱建議：

- 導覽文字：月報表
- 頁面標題：月帳務報表
- JS 檔案：`js/monthly-report.js`

維持目前最多一層資料夾的結構，不新增更深層資料夾。

## 3. 資料來源

月帳務報表使用既有 `data.transactions`。

納入統計：

- `type: "expense"`：支出報表
- `type: "income"`：收入報表

不納入統計：

- `transfer-in`
- `transfer-out`
- 舊資料若仍有 `adjustment`，不列入收入 / 支出報表

分類來源：

- 優先使用交易的 `category`
- 若沒有分類，顯示為「未分類」

金額來源：

- 使用 `transaction.amount`
- 支出與收入頁各自顯示正數總額，不在圖表中顯示負號

## 4. 頁面結構

參考圖片後，頁面由上到下分成：

1. 收入 / 支出切換
2. 日期範圍列
3. 期間切換
4. 環形分類圖
5. 分類百分比摘要
6. 分類明細清單

### 4.1 收入 / 支出切換

頂部使用 segmented control：

```text
支出 | 收入
```

預設顯示「支出」。

切換後：

- 圖表總額文字改成「總支出」或「總收入」
- 分類摘要與明細重新計算
- 日期範圍不變

### 4.2 日期範圍列

顯示目前統計範圍：

```text
2026/05/01 ~ 2026/05/31
```

左右放上一期 / 下一期按鈕：

- 左箭頭：切到上一個期間
- 右箭頭：切到下一個期間

範圍格式：

- 月：該月第一天到該月最後一天
- 半年：連續六個月
- 年：該年 1/1 到 12/31
- 自訂：由使用者選開始日與結束日

### 4.3 期間切換

使用 segmented pills：

```text
月 / 半年 / 年 / 自訂
```

第一版建議先完整實作：

- 月
- 半年
- 年

自訂可以先做簡單版：

- 顯示開始日期與結束日期 input
- 使用者改日期後立即更新統計

## 5. 圖表設計

### 5.1 環形圖

使用純前端 CSS / JS，不引入圖表套件。

建議做法：

- 用 CSS `conic-gradient()` 畫環形圖
- 中間用白色圓形覆蓋形成 donut
- 中間顯示：

```text
總支出
$43,318
```

或：

```text
總收入
$80,000
```

若該期間沒有資料：

- 顯示灰色空環
- 中間顯示 `$0`
- 分類摘要顯示空狀態

### 5.2 分類顏色

每個分類固定對應一個顏色。

建議先建立一組顏色循環：

```text
粉紅、黃色、橘色、綠色、藍色、紫色、灰色、青色
```

同一分類在收入 / 支出、不同月份中盡量保持相同顏色。

## 6. 分類摘要

圖表下方顯示分類百分比摘要，參考圖片的雙欄樣式。

每個分類項目顯示：

```text
[色塊] 分類名稱  67.6%
```

排序規則：

1. 金額高到低
2. 金額相同時依分類名稱排序

第一版摘要最多顯示前 6 個分類。

若分類超過 6 個：

- 圖表仍包含全部分類
- 摘要先只顯示前 6 個
- 明細清單顯示全部分類

## 7. 明細清單

下方區塊標題依模式顯示：

- 支出模式：支出明細
- 收入模式：收入明細

每列顯示：

```text
[分類圓形圖示] 分類名稱       $金額
```

第一版不需要真的做複雜 icon，可以先用分類顏色圓點或簡單文字符號。

點擊分類列的後續可擴充：

- 顯示該分類底下的交易紀錄

但第一版可以先不做分類展開，避免一次太複雜。

## 8. 統計邏輯

新增 `js/monthly-report.js`，負責：

- 報表狀態
  - `mode`: `expense` / `income`
  - `period`: `month` / `half-year` / `year` / `custom`
  - `anchorDate`
  - `customStartDate`
  - `customEndDate`
- 日期範圍計算
- 交易篩選
- 分類彙總
- 百分比計算
- 圖表 HTML 產生
- 明細 HTML 產生
- 事件綁定

核心計算：

```text
分類金額 = 該期間、該模式、該分類的交易 amount 加總
總額 = 所有分類金額加總
分類百分比 = 分類金額 / 總額 * 100
```

## 9. 預計修改檔案

實作時預計修改：

```text
index.html
css/style.css
js/app.js
service-worker.js
implementation plan.md
```

實作時預計新增：

```text
js/monthly-report.js
```

若實作完成後要離線可用：

- `service-worker.js` 需加入 `./js/monthly-report.js`
- 快取版本需升版

## 10. UI 注意事項

- 報表頁以手機優先設計。
- 不使用大型說明文字。
- 不做 landing page。
- 圖表不要放在過度裝飾的卡片中，保持像 App 內建統計頁。
- 日期切換、模式切換、期間切換都要容易用手指點。
- 長分類名稱要能換行或截斷，不可擠壓金額。
- 金額需使用既有 `formatCurrency()`。

## 11. 第一版先不做

為了控制複雜度，第一版先不做：

- 匯出報表 PDF
- 長期趨勢折線圖
- 分類點擊展開交易明細
- 自訂分類顏色
- 多頁 legend 分頁圓點
- 圖表動畫

## 12. 驗證計畫

實作後需要驗證：

- `node --check` 通過
- 月 / 半年 / 年 / 自訂範圍計算正確
- 收入 / 支出切換後總額正確
- 分類百分比加總接近 100%
- 沒有交易時不出錯
- service worker 能載入新增檔案
- 本機靜態載入 `index.html`、`js/monthly-report.js`、`css/style.css` 成功
